# Dense N-Sweep CWS 8M GPU Results

Source CSV: `artifacts/cws_balanced_branch_8m_gpu_event_conditioned_dense_n_sweep/n-sweep.csv`

This report tests the large-`N` claim in the stricter form:

```text
WPU should scale with causal working set K, not total state N, when K is small and identifiable.
```

Setup: RTX 5070 Ti, PyTorch 2.11.0+cu128, hidden_dim=512, layers=2,
balanced branch labels, fixed causal working set K=8, event-conditioned WPU
propagation, and five seeds per condition. The sweep covers eight world sizes:
N = 64, 128, 256, 512, 1024, 2048, 4096, and 8192.

Main reading: the result does not support a monotonic "larger N always makes
WPU better" claim. Accuracy is noisy, and the best WPU variant changes between
oracle and learned selection. The stronger and more stable result is a
compute-regime claim: token and graph baselines show rapidly increasing latency
with N, while WPU latency grows more slowly because it updates a bounded causal
working set. At N=4096, oracle WPU is both more accurate and faster than the
baselines. At N=8192, WPU variants remain above the baselines in accuracy and
are much faster, but the confidence intervals are still large.

Interpretation: this is positive evidence for a large-N/small-K regime, not a
universal dominance result. The next experiment should reduce WPU variance by
improving selector supervision and branch calibration, then repeat the same
N-sweep with harder distractors.
