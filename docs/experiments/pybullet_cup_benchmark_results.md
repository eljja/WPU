# PyBullet Cup Benchmark

This experiment is the first simulator-grounded WPU benchmark in the repository.
Samples are generated by a PyBullet rigid-body rollout, converted into explicit
`WorldState` objects, and then evaluated through the same `StateGraphBatch`
interface used by synthetic WPU experiments.

The goal is not to claim real-world physics understanding. The goal is narrower:
verify that WPU can process objectified state emitted by an independent simulator
and test whether pre-tensor indexed retrieval decouples WPU latency from
irrelevant background state.

## Protocol

- Simulator: PyBullet `DIRECT` rigid-body rollout.
- Scenario: a cup on a table receives a hand impulse near a table edge.
- Branch labels: `stable`, `falls`, `caught/recovered`.
- Labels: balanced by rejection sampling.
- WPU input: pre-tensor indexed event-local subgraph.
- Baseline input: full simulator-derived state graph.
- Models: `wpu-cws-indexed-sparse`, `wpu-cws-indexed-local-dense`,
  `graph-transformer`, `serialized-token`.
- Seeds: `11, 13, 17, 19, 23`.
- Background objects: `0, 32, 128`.
- Training: 20 steps, batch 8, hidden 64.
- Evaluation: 36 samples per condition.

## Interpretation

This result supports a limited systems claim: the indexed WPU path keeps the
neural working set near `K ~= 4-5` even as full simulator state grows with
background objects. The graph-transformer baseline slows sharply because it
processes the full state graph. This is consistent with the WPU regime claim:
large `N`, small identifiable `K`, and event-local propagation.

The result does not yet support an accuracy-dominance claim. On this small
benchmark, WPU accuracy is comparable to token/graph baselines but not clearly
better. The serialized-token baseline is also very fast at this small scale in
the current PyTorch implementation, so WPU's advantage is not universal latency
dominance either.

## Issues Found

- The first version of the benchmark was label-imbalanced, with majority
  accuracy near 0.83. It was replaced by balanced rejection sampling.
- The first version unfairly projected the state for all models. It now applies
  pre-tensor indexed retrieval only to WPU indexed models; baselines receive the
  full simulator state.
- `caught` labels did not occur under the original final-state rule. The label
  now treats catcher-enabled, large-displacement, non-fall outcomes as
  `caught/recovered`.

## Next Steps

- Increase seeds, training steps, and simulator sample count.
- Add relation/objectification corruption to test perception-like errors.
- Add larger `N` without forcing PyBullet to simulate all background bodies;
  background objects should be simulator-exported but optionally frozen.
- Add matched-parameter token and graph baselines with the same full-state
  information budget.
- Add long-horizon closed-loop PyBullet rollouts where model-predicted deltas
  are applied back into `WorldState`.
